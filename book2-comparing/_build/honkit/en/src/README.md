# The Harness Design Philosophies of Claude Code and Codex

![Cover: The Harness Design Philosophies of Claude Code and Codex](assets/cover-wxb-en.svg)

Subtitle: convergence, divergence, and the engineering choices behind two coding-agent systems.

> “Life is suffering, and harness engineering is the art of making suffering deliberate.”

This book is a comparative anatomy of two harnesses: Claude Code, which grew out of operational experience, and Codex, which was born from a highly explicit policy and control layer. We are not tallying tools; we are examining where each system offloads trust, structures governance, and constrains a fundamentally unreliable model.

Three assumptions that guide this story:

- The comparison is centered on harness design, not model capabilities.
- A harness is a deliberate power-distribution system, not a feature checklist.
- Engineering differences usually emerge from where order is placed, not what it is called.
